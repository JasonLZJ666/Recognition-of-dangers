# 危险标志识别系统设计报告

## 项目概述

本项目围绕复杂环境下的危险标志识别任务，完成数据素材整理、模型方案设计、系统界面开发和设计报告编写。系统交付包含 5 类危险标志样本、苹果官网式极简浏览器端识别系统，以及用于后续正式训练的增强版深度学习训练脚本。

## 需求对应

| 项目要求 | 完成情况 |
|---|---|
| 自主采集数据，危险标志图片见附件 | 已从 PDF 素材中提取 5 张标准危险标志图片，并按类别整理为数据集 |
| 完成模型选型并完成模型训练、优化 | 已给出 EfficientNet-B0、ResNet18、MobileNetV3 和 StrongCNN 可选训练脚本，包含增强、调度、早停和评估输出 |
| 完成系统开发，可以检测给定图片、视频或摄像头中的危险标志 | 前端系统支持图片、视频、摄像头三种输入源 |
| 完成登录界面和系统检测界面设计 | 已实现苹果官网式极简登录页、检测控制台、结果面板和样本库 |
| 撰写设计报告 | 已生成正式 DOCX 设计报告 |

## 数据集建设

PDF 附件共包含 5 张危险标志素材，已整理为 `dataset/<class_name>/` 目录。当前样本属于标准图标数据，适合完成原型验证；若要部署到真实工地、道路或仓储环境，应继续补充不同角度、光照、遮挡和背景下的现场图片。

## 模型选型与优化

项目训练管线支持 EfficientNet-B0、ResNet18、MobileNetV3-Small 和 StrongCNN 四种结构。默认推荐 EfficientNet-B0 迁移学习方案：利用 ImageNet 预训练特征提取能力，并通过冻结骨干、替换分类头、Label Smoothing、AdamW 和 CosineAnnealingLR 提升小样本场景下的稳定性。自定义 StrongCNN 采用深度可分离卷积、残差连接和 Squeeze-Excitation 通道注意力，在无法下载预训练权重时仍可展示自主设计的 CNN 结构。训练脚本还会输出训练历史、混淆矩阵、配置 JSON、最佳权重和最终权重，便于展示完整实验过程。

训练脚本通过随机仿射、透视扰动、颜色扰动、RandAugment 和随机擦除增强样本，提升模型对拍摄角度、亮度变化、遮挡和轻微变形的鲁棒性。训练指标从单一准确率扩展为 Top-1、Top-3、宏平均 F1、加权 F1、逐类别 Precision/Recall/F1，并输出 dataset_audit、experiment_report、model_card 和 classification_report。训练代码按工程模块拆分为 constants、config、datasets、architectures、metrics、callbacks、model_utils、reporting、data_audit、training_engine、evaluate_model、train_model 和 export_onnx，便于维护、答辩展示和后续扩展。

浏览器端已导入训练模型：PyTorch 权重通过 export_onnx.py 转换为 ONNX，并由 ONNX Runtime Web 在前端完成推理。系统同时保留无依赖离线相似度识别作为回退方案，确保模型文件或运行库不可用时仍可完成演示检测。

## 系统设计

系统采用前端单页应用结构，包含登录模块、输入源管理模块、识别模块、结果展示模块、样本库模块和训练方案展示模块。界面改为苹果官网式极简风格，以浅灰页面、白色产品卡片、大标题、细分割线、低阴影和蓝色强调按钮为主，整体更轻、更扁平、更精致。图片检测通过文件读取完成；视频检测按固定帧率抽帧识别；摄像头检测调用浏览器媒体接口并实时绘制检测框。

## 测试结论

使用 PDF 提取的 5 张标准样本进行验收测试时，系统能够正确识别对应类别，并显示置信度、风险等级和处置建议。由于当前样本量较少，真实复杂环境中的泛化能力依赖后续数据扩充和 CNN 权重训练。

## 使用说明

打开 `app/index.html` 后使用默认账号 `admin`、密码 `123456` 登录。进入控制台后选择图片、视频或摄像头模式即可检测。摄像头模式需要浏览器授权摄像头权限。
