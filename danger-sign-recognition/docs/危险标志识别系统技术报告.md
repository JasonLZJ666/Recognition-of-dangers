# 危险标志识别系统技术报告

## 摘要

本项目面向危险警示标志自动识别场景，设计并实现了一套包含数据集构建、数据增强、模型训练、模型评估、Python 后端推理和网页端交互的完整危险标志识别系统。系统支持对上传图片、拖拽图片和摄像头画面进行识别，并输出预测类别、置信度、风险等级、处置建议和推理耗时。

当前版本采用 Python 前端推理方案。浏览器页面负责采集图片或摄像头画面，Python 后端在服务启动时自动加载已经训练好的 PyTorch 模型并完成推理。该方案避免了浏览器 ONNX Runtime Web 和 WASM 后端兼容问题，适合课程设计、答辩演示和本地部署。

模型部分采用 EfficientNet-B0 迁移学习结构，识别 5 类危险标志：爆炸风险、高处坠物、落石危险、易燃危险和水域危险。项目通过旋转、缩放、透视、亮度、对比度、模糊、阴影和背景扰动等方式构建 `dataset_viewpoint/` 视角与光照增强数据集，以提升模型对真实拍摄条件的适应能力。当前推荐模型在独立增强测试集上的 Top-1 准确率为 0.9975，Macro-F1 为 0.9975；在前端测试照片集上的识别结果为 30/30。

关键词：危险标志识别；图像分类；EfficientNet-B0；PyTorch；迁移学习；数据增强；Python Web 前端；模型评估

## 1. 项目背景

危险警示标志广泛出现在施工现场、实验室、仓储区域、道路边坡、水域边界和公共安全管理场景中。传统人工识别方式依赖人员观察，容易受到疲劳、视角遮挡、光照变化和巡检效率的影响。随着计算机视觉和深度学习的发展，可以利用图像分类模型辅助识别危险标志，为安全巡检和风险提示提供支持。

本项目围绕课程设计和验收演示需求，完成以下工作：

- 从附件素材中整理 5 类危险标志基础数据。
- 构建视角和光照增强数据集，模拟不同拍摄条件。
- 采用 EfficientNet-B0 迁移学习训练 5 分类模型。
- 实现训练历史、模型配置、混淆矩阵、模型卡片和评估报告导出。
- 实现 Python Web 前端，启动时自动加载训练好的 `.pt` 模型。
- 支持图片上传、拖拽图片、摄像头单帧识别和实时识别。
- 编写 README、部署说明、技术报告、答辩 PPT 和答辩演讲稿。

## 3. 系统总体架构

当前系统采用浏览器加 Python 后端的本地 B/S 架构。整体流程如下：

```text
浏览器页面
  ├─ 图片上传
  ├─ 拖拽图片
  ├─ 摄像头采集
  └─ 结果展示
        ↓ HTTP JSON
Python Web 后端
  ├─ 接收 base64 图片
  ├─ 图像解码与 EXIF 方向修正
  ├─ 危险标志区域定位与裁剪
  ├─ 图像缩放与归一化
  ├─ PyTorch 模型推理
  └─ 返回类别、置信度、裁剪框和耗时
        ↓
EfficientNet-B0 分类模型
```

核心文件如下：

```text
python_frontend/server.py                  Python Web 服务与推理接口
python_frontend/start_python_frontend.ps1  一键启动脚本
model/artifacts_viewpoint/best_danger_sign_model.pt  当前推荐模型
model/architectures.py                     模型结构定义
model/train_model.py                       训练入口
model/evaluate_model.py                    测试集评估
model/evaluate_frontend_photos.py          前端照片评估
```

该架构的主要优点是推理环境可控。浏览器不直接加载模型，也不依赖 ONNX Runtime Web 的 WASM 后端；模型推理统一在 Python 后端完成，减少了浏览器差异导致的加载失败问题。

## 5. 图像预处理流程

训练阶段和推理阶段都需要进行图像预处理。

### 5.1 训练阶段增强

训练代码中使用 `model/datasets.py` 定义数据变换。训练阶段主要变换包括：

- `Resize`
- `RandomResizedCrop`
- `ColorJitter`
- `RandomAffine`
- `RandomPerspective`
- `RandAugment`
- `RandomErasing`
- ImageNet 均值方差归一化

这些变换用于提升模型对光照、角度、尺度、轻微遮挡和图像变形的鲁棒性。

### 5.2 推理阶段预处理

Python 前端推理时，后端会执行以下步骤：

1. 接收前端传来的 base64 图片。
2. 使用 PIL 解码图片。
3. 修正 EXIF 方向，避免手机照片旋转错误。
4. 转换为 RGB 图像。
5. 根据黄色区域和深色图形区域定位危险标志主体。
6. 对定位区域进行正方形裁剪并增加一定 padding。
7. 缩放到模型输入尺寸 224 x 224。
8. 使用 ImageNet 均值和方差归一化。
9. 输入 PyTorch 模型进行推理。

该流程保证前端实际输入与训练阶段的数据分布尽量一致。

## 6. 模型设计

### 6.1 模型选型

项目当前推荐模型为：

```text
EfficientNet-B0
```

选择原因如下：

- 参数规模较小，适合本地 CPU 或普通电脑运行。
- 分类性能较好，适合 5 类图像分类任务。
- 支持 ImageNet 预训练权重，便于小样本迁移学习。
- 在模型体积、准确率和推理速度之间比较均衡。

训练代码中还保留了其他结构：

```text
efficientnet_b0
resnet18
mobilenet_v3_small
strong_cnn
```

其中 `strong_cnn` 是自定义卷积网络，包含深度可分离卷积、残差连接和 Squeeze-Excitation 通道注意力，可用于展示自主模型结构。

### 6.2 模型结构与分类头

EfficientNet-B0 保留主干特征提取能力，替换末端分类头。分类头结构包括：

- Dropout
- Linear
- SiLU 激活
- Dropout
- Linear 输出 5 个类别 logits

当前模型参数统计如下：

| 项目 | 数值 |
|---|---:|
| 总参数量 | 4,336,769 |
| 可训练参数量 | 329,221 |
| 冻结参数量 | 4,007,548 |
| 可训练比例 | 7.59% |
| 估计模型大小 | 16.704 MB |
| 输入尺寸 | 224 x 224 |

冻结主干的原因是基础数据较少，完全训练所有参数容易过拟合。冻结主干后主要训练分类头，可以降低训练成本，并提高小样本任务的稳定性。

### 7.3 优化策略

训练过程采用以下策略：

- AdamW 优化器：改善权重衰减处理，提高训练稳定性。
- CosineAnnealingLR：使用余弦退火调度学习率。
- Label Smoothing：降低模型对训练样本的过度自信。
- Dropout：减少分类头过拟合。
- Early Stopping：验证指标长期不提升时提前停止。
- Macro-F1 作为最佳模型指标：避免只看总体准确率而忽略类别均衡。

## 8. 模型评估方法

这一节回答“怎么评估模型”。

### 8.1 评估独立测试集

第一步，生成独立增强测试集：

```powershell
python model\build_viewpoint_dataset.py --source dataset --out dataset_test_viewpoint --per-class 80 --size 320 --seed 9090 --no-source-copy
```

第二步，运行评估脚本：

```powershell
python model\evaluate_model.py --checkpoint model\artifacts_viewpoint\best_danger_sign_model.pt --dataset dataset_test_viewpoint --out model\artifacts_viewpoint\test_classification_report.json --repeats 1
```

第三步，查看命令行输出：

```text
classification report: model\artifacts_viewpoint\test_classification_report.json
top1=0.9975 top3=1.0000 macro_f1=0.9975
```

第四步，查看详细 JSON 报告：

```text
model\artifacts_viewpoint\test_classification_report.json
```

该文件包含：

- `top1`：Top-1 准确率。
- `top3`：Top-3 准确率。
- `macro_precision`：宏平均精确率。
- `macro_recall`：宏平均召回率。
- `macro_f1`：宏平均 F1。
- `weighted_f1`：加权 F1。
- `confusion_matrix`：混淆矩阵。
- `per_class`：每个类别的 Precision、Recall 和 F1。

### 8.2 评估前端照片

前端照片评估命令如下：


### 8.3 指标解释

| 指标 | 含义 | 本项目用途 |
|---|---|---|
| Top-1 | 模型最高置信度类别是否正确 | 主要准确率指标 |
| Top-3 | 正确类别是否在前三个候选中 | 观察模型候选排序能力 |
| Precision | 预测为某类的样本中有多少是真的 | 检查误报 |
| Recall | 某类真实样本中有多少被识别出来 | 检查漏报 |
| F1 | Precision 与 Recall 的综合指标 | 更全面评价分类效果 |
| Macro-F1 | 各类别 F1 的平均值 | 适合类别均衡评价 |
| Confusion Matrix | 展示类别之间的混淆情况 | 定位易混类别 |

`/api/status` 返回信息包括：

- 模型是否加载。
- checkpoint 路径。
- 模型结构。
- 推理设备。
- 类别列表。
- 输入图像尺寸。
- 最佳 epoch 和最佳指标。

`/api/predict` 返回信息包括：

- `best`：最高置信度类别。
- `candidates`：5 类候选结果及置信度。
- `crop`：自动裁剪框。
- `latencyMs`：推理耗时。
- `model`：当前模型信息。

## 14. 不足与改进方向

当前系统可以满足课程设计和本地演示需求，但仍有以下不足：

1. 数据来源仍然有限。当前数据主要来自附件和程序增强，真实现场照片数量较少。
2. 当前任务是图像分类。如果一张图片中出现多个危险标志，分类模型无法分别定位每个标志。
3. 当前部署方式依赖 Python 环境。其他电脑需要安装 PyTorch 等依赖。
4. 摄像头实时识别仍可加入连续帧投票，减少偶发误判。
5. 后续可以扩展为目标检测模型，例如 YOLO 系列，以支持多目标定位和识别。

后续优化方向包括：

- 采集更多真实场景图片。
- 引入目标检测模型。
- 增加连续帧稳定策略。
- 将 Python 前端封装为可执行文件或 Docker 镜像。
- 增加更详细的错误日志和可视化调试页面。

## 15. 结论

本项目完成了一套面向危险警示标志识别的完整系统。系统从数据整理出发，通过视角和光照增强构建训练集，采用 EfficientNet-B0 迁移学习完成 5 类危险标志分类，并通过 Python Web 前端实现图片和摄像头识别功能。

当前模型在独立增强测试集上的 Top-1 准确率为 0.9975，Macro-F1 为 0.9975；在前端测试照片集上达到 30/30。实验结果说明，当前数据增强、模型训练和前端推理链路能够满足课程设计、答辩演示和本地验收需求。

与浏览器 ONNX 前端相比，Python 前端版本启动时直接加载 PyTorch `.pt` 模型，推理链路更稳定，避免了浏览器 WASM 兼容问题。因此，当前 Python 前端推理版适合作为项目最终交付版本。
